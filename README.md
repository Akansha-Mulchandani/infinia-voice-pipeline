# infinia-voice-pipeline

Open-source voice cloning TTS pipelines for English, Arabic (MSA), and Hindi with comprehensive evaluation harness. This project benchmarks multiple zero-shot voice cloning models across three languages using six objective metrics (MOS, speaker similarity, latency, RTF, WER) to identify the best-performing model per language for production deployment.

## Setup & Reproduction

### Prerequisites

- Python 3.10+
- GPU with 16GB VRAM (tested on Kaggle T4 x2)
- Reference audio: 15-20 seconds of clean mono WAV at 16kHz+

### Local Development (Windows/Linux/Mac)

1. Clone the repository:
```bash
git clone https://github.com/<YOUR_USERNAME>/infinia-voice-pipeline.git
cd infinia-voice-pipeline
```

2. Install dependencies:
```bash
pip install -r requirements.txt
```

3. Add your reference audio:
```bash
# Convert your audio to mono 16kHz WAV
ffmpeg -i your_audio.m4a -ac 1 -ar 16000 data/reference_audio/me.wav
```

4. Run a pipeline:
```bash
python pipelines/english/run_chatterbox.py --ref data/reference_audio/me.wav
```

### Kaggle Execution (Recommended)

For actual model inference and benchmarking, use Kaggle's free GPU notebooks. See **KAGGLE.md** for exact step-by-step instructions.

## Failure Modes

### Known Limitations

1. **MMS-TTS (Arabic)**: Does not support zero-shot voice cloning. Output uses pre-trained voice, not reference speaker. Speaker similarity metrics will be meaningless.

2. **Indic Parler-TTS (Hindi)**: May not support voice cloning depending on the specific model version. Some Indic TTS models are single-speaker only.

3. **XTTS-v2 Language Support**: Language support varies by version. Arabic and Hindi support needs verification in the installed version.

4. **CosyVoice2 Installation**: Requires git install from FunAudioLLM/CosyVoice repository. Installation may fail if the repository structure changes.

5. **Chatterbox API**: The chatterbox-tts package API may differ from the implementation. Needs verification on actual installation.

### Expected Failure Modes

- **Arabic XTTS-v2**: May mispronounce numbers above three digits or complex numerals
- **Hindi models**: May drop reference speaker's pitch on long sentences (>20 words)
- **All models**: May struggle with proper nouns not in training data
- **Streaming latency**: Actual streaming latency may differ from batch synthesis time

## What's Missing / Future Improvements

### Current Gaps

1. **MOS Collection**: Requires human listeners. Currently a placeholder - needs actual subjective ratings from 2-3 listeners per language.

2. **Model API Verification**: All model APIs (Chatterbox, CosyVoice2, Indic Parler-TTS) need verification against actual installed versions. The implementations are based on documentation and may need adjustment.

3. **Language Code Verification**: XTTS-v2 language codes for Arabic and Hindi need verification in the installed version.

4. **Bonus Indic Language**: Brief mentions trying one more Indic language (Tamil/Bengali/Marathi). Not implemented yet.

### Potential Improvements

1. **More Models**: Add Fish-Speech for Arabic if it supports the language and voice cloning.

2. **Better MOS Collection**: Implement a web-based MOS collection interface for easier rating.

3. **Automated Model Selection**: Implement a per-language router that automatically selects the best model based on text characteristics.

4. **Streaming Latency Measurement**: Implement actual streaming latency measurement (time to first audio chunk) instead of batch synthesis time.

5. **Cross-Lingual Voice Cloning**: Test cross-lingual capabilities (e.g., cloning English reference for Hindi synthesis).

6. **Fine-tuning**: Investigate fine-tuning models on specific reference voices for better similarity.

## Evaluation Metrics

The evaluation harness measures 6 metrics per pipeline:

1. **MOS (Mean Opinion Score)**: Subjective quality rating (1-5), target ≥4.0
2. **Speaker Similarity**: Cosine similarity of speaker embeddings, target ≥0.75
3. **Latency**: Time to generate audio (median of 5 runs), target <500ms streaming / <2s batch
4. **RTF (Real-Time Factor)**: gen_time / audio_duration, target ≤0.5
5. **WER (Word Error Rate)**: Round-trip transcription accuracy, target ≤10%
6. **Cross-Language Consistency**: Flags if any language misses targets
